GitHub Token

What you get with a token:
- API rate limit increases from 60 to 5,000 requests/hour
- CI build detection — checks GitHub Actions artifacts alongside releases and pre-releases
- CI artifact downloads — install CI builds directly from the WebUI
- Per-module toggle to include/exclude CI builds

The module works without a token — you just won't see CI builds.

How to get a token:
1. Go to github.com/settings/tokens
2. Click "Generate new token (classic)"
3. Name it (e.g. "module-update-checker")
4. Check public_repo scope (required for CI artifact downloads)
5. Generate and paste it here
Token is stored in plaintext at /data/adb/muc/token (chmod 600). Any root app can read it. No scopes = can only read public repos, but it's tied to your GitHub account.
